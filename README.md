*[School-Library-System-with-ruby]* 

A Ruby-based console application for managing a school library system. This project demonstrates core Object-Oriented Programming (OOP) principles including encapsulation, inheritance, and polymorphism by modeling real-world entities like `Person`, `Student`, and `Teacher`

## 🛠 Built With <a name="built-with"></a>

 *Ruby*


### Tech Stack <a name="tech-stack"></a>

- **Ruby**
- **OOP Principles**
  - Encapsulation (private methods and restricted access)
  - Inheritance (using `class Child < Parent`)
  - Method overriding (e.g., `can_use_services?`)
- Default parameters and constructors
- Gitflow workflow
- Ruby code best practices and linter compliance

### Key Features <a name="key-features"></a>

- Define a `Person` class with attributes like name, age, and parent permission.
- Extend `Person` to build `Student` and `Teacher` classes with unique behaviors.
- Override and extend methods to simulate real-world roles and access control.
- Well-structured and modular Ruby codebase — each class is saved in a separate file.
